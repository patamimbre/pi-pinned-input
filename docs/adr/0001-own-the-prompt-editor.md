# Own the prompt editor through Pi's extension API

The extension will install through Pi's public `setEditorComponent` API so it can implement bounded auto-growth and OpenCode-style prompt-history navigation. It wraps an existing custom editor when that editor exposes Pi's standard layout and cursor methods, preserving extensions such as Amp Editor; incompatible editors fail clearly rather than being replaced. This deliberately rejects copying or patching `Editor` private fields, though a small version-checked adapter remains necessary while Pi hardcodes its viewport ratio: the adapter checks Pi's exported `VERSION` against the tested releases and keeps Pi's default prompt editor (with a visible warning) on untested versions. Wrapped editors keep their own prompt-history behavior; the extension adds bounded growth and draft-boundary cursor redirection on top.
